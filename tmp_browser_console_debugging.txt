why the coop policy appear
The message
Cross-Origin-Opener-Policy policy would block the window.closed call.
comes from the browser's Cross-Origin-Opener-Policy (COOP) security feature.
What is COOP?
COOP is an HTTP response header that isolates a webpage from pages opened from other origins
(domains). It helps protect against attacks such as Spectre and prevents cross-origin pages from
interfering with each other.
For example, a website might send:
http
Cross-Origin-Opener-Policy: same-origin
or
http
Cross-Origin-Opener-Policy: same-origin-allow-popups
Why does this message appear?
A very common ﬂow is:
1. Kling opens a popup window for login/payment/OAuth.
2. The popup ﬁnishes.
3. Some JavaScript tries to execute:
JavaScript
window.close();
or
JavaScript
window.opener.close();
4. Because the popup and the main page are considered diﬀerent browsing contexts under COOP,
Chrome blocks that operation and prints:
Cross-Origin-Opener-Policy policy would block the window.closed call.
It's a warning from Chrome, not necessarily an application error.
